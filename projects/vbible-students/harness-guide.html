3개 레이어로 분리된 하네스

"통합" 원칙: 같은 도메인 하네스를 두 개 만들지 않는다. 데이터와 메커니즘 정본을 한 곳으로 모으고, V-Bible 레이어는 그것을 가리키는 얇은 인덱스로 둔다. 이것이 드리프트(불일치)를 막아 지속가능성을 확보한다.

1

단일 진실 원천 — 데이터 · V-Bible 메커니즘

projects/vbible-students/assets/reference-media.json — 5인 캐릭터의 elementId·voiceId·characterSheet와 meta.*(Elements·립싱크·한국어 TTS·스타일락) 정본. 값을 복사하지 않고 항상 새로 읽는다.

2

범용 엔진 — higgsfield-shortform 플러그인

범용 숏폼 제작 메커니즘(키프레임·렌더·일관성). short-style/short-render/short-consistency 스킬을 Elements·한국어 TTS·wan2_7 립싱크 지원으로 업그레이드해 재사용.

3

V-Bible 얇은 레이어 — 프로젝트 전용

vbible-foundation(헌법·불변식 인덱스) + vbible-pipeline(7단계 오케스트레이터) + 에이전트 5종 + lib/state.py(게이트 상태머신). 두 정본을 가리키는 얇은 설정·강제 레이어.

7단계 파이프라인

에피소드 한 편이 만들어지는 흐름

각 단계를 눌러 펼쳐 보세요. 게이트 = 사용자 승인 필요, 비용 = 크레딧 소모(생성 전 승인), 자동 = 게이트 없음. QA는 2·3·5단계 직후 경계면을 점검합니다.

1 intake · 대본 수용사용자 대본(문서·문장) 수용·정규화 게이트
사용자가 준 대본을 출발점으로 삼습니다. 주제를 창작하지 않습니다.
  • 완성 대본(문서/긴 텍스트) → 원문 존중, 한국어 정규화·화자 표기만 정돈
  • 짧은 문장/로그라인 → 한국어 대본으로 확장
  • 성경 검증(theology-reviewer) 통과 · 산출물 briefs/script.md
담당 · vbible-scriptwriter → vbible-theology-reviewer
2 breakdown · 씬 분할대본 → 씬 · 화자분리 · 샷리스트 게이트
승인된 대본을 씬으로 나누고, 립싱크용으로 대사를 화자별 단독 컷으로 분리합니다. 컷 수는 대본 길이에 따라 가변.
  • 대본 속 화자를 5인 캐릭터에 매칭(내레이션만이면 오프스크린 트랙)
  • 사용자 원문 대사는 표현 보존 · QA 경계 점검 통과해야 진행
  • 산출물 briefs/shotlist.md
담당 · vbible-scriptwriter → vbible-qa
3 style · 키프레임Elements 인물고정 키프레임 생성 게이트비용

  • 기본 모델 nano_banana_2, 9:16, 화면 내 글자 금지

절대 불변식

위반하면 = 작업 미완료

매 편 강제되는 5가지 규칙. 세부 메커니즘·고정값의 정본은 reference-media.json meta.*이며, 아래는 규칙 요약입니다.

1 · 단일 진실 원천

캐릭터·voiceId·정책은 reference-media.json에만. 본문에 복사 금지, 항상 새로 읽기.

2 · 한국어 대사 필수

모든 대사·나레이션 한국어. 영상 모델 자동 오디오 금지 → ElevenLabs TTS 별도 트랙.

3 · Elements 인물 고정

<<<elementId>>> 임베드로 얼굴·의상 고정. Soul 미사용, 다중 인물 = placeholder 여러 개.

4 · 립싱크 · 무음 i2v

립싱크 wan2_7 화자 1명/컷, i2v seedance_2_0 무음, 키프레임 nano_banana_2. 화면 내 글자 금지.

5 · 예산 게이트

편당 ≤650 크레딧, 월 3,000. 모든 생성 전 비용 고지·승인 — 승인 없는 지출 금지.

5인 캐릭터

반복 등장 가족 · 친구

핵심 식별자는 절대 변경 금지(forbiddenVariations). 상세·고정값은 reference-media.json.

시작하기

엔진 플러그인을 한 번 설치한 뒤, 자연어로 에피소드 제작을 요청하면 vbible-pipeline이 7단계를 진행합니다.

① 엔진 설치 (최초 1회)

# 슬래시 커맨드로 직접 입력
/plugin marketplace add ../my_higgsfield
/plugin install higgsfield-shortform

② 대본 제공 → 자동 제작

# 대본 파일로
"이 대본으로 영상 만들어: ep1.md"
# 또는 문장/텍스트로
"하은이 기도를 묻는 짧은 장면 영상"

③ 재개 · 부분 재실행

# 중단 지점부터 이어서
"vbible-ep-01 재개"
# 특정 단계만 다시
"키프레임만 다시" · "렌더 재실행"
